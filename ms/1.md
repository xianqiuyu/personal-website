## 节流函数（throttle）标准实现整理

下面是整理后的说明与代码，并修复了原文中 `cancel` 无法访问内部变量的问题。

### 思路说明

- **目标**：在高频触发的场景（如 `scroll`、`resize`、按钮点击）中，限制真正执行函数的频率。
- **关键点**：
  - 使用 `timer` 控制在一段时间内只触发一次。
  - 可选支持**立即执行一次**（`immediate = true` 时）。
  - 返回的函数挂载 `cancel` 方法，用来：
    - 取消还未执行的定时任务；
    - 重置内部状态，下一次可以重新“立即执行”。

### 实现代码（已支持 cancel）

```javascript
// 节流函数
function throttle(fn, delay, immediate = false) {
  let timer = null;
  let isInvoked = false; // 标记是否已经进行过“立即执行”

  function throttled(...args) {
    const context = this;

    // 1. 立即执行：只在第一次触发时生效
    if (immediate && !isInvoked) {
      fn.apply(context, args);
      isInvoked = true;
      return;
    }

    // 2. 如果当前已经在等待下次执行，直接返回
    if (timer) return;

    // 3. 延迟执行（尾触发）
    timer = setTimeout(() => {
      fn.apply(context, args);
      timer = null;
    }, delay);
  }

  // 提供取消方法：清除定时器并重置立即执行标记
  throttled.cancel = () => {
    if (timer) {
      clearTimeout(timer);
      timer = null;
    }
    isInvoked = false;
  };

  return throttled;
}

// 使用示例
const onScroll = () => {
  console.log('scroll handler:', Date.now());
};

// 1000ms 内最多执行一次，且第一次触发会立即执行
const throttledScroll = throttle(onScroll, 1000, true);

window.addEventListener('scroll', throttledScroll);

// 需要时可主动取消
// throttledScroll.cancel();
```

### 代码正确性检查说明

- **闭包变量可见性**：
  - `timer` 与 `isInvoked` 定义在 `throttle` 作用域中，同时被 `throttled` 与 `throttled.cancel` 引用，确保：
    - `cancel` 能访问并清除当前的定时器。
    - 重新设置 `isInvoked = false` 后，下一次在 `immediate = true` 时会重新立即执行。
- **行为特征**：
  - 高频触发时：
    - 若 `immediate = true`：第一次立即执行，之后在每个 `delay` 间隔内最多执行一次（尾触发）。
    - 若 `immediate = false`：在每个 `delay` 间隔内最多执行一次（纯尾触发）。
  - 调用 `throttled.cancel()` 后：
    - 当前等待执行的调用被取消；
    - `isInvoked` 被重置，下一次在 `immediate = true` 时仍会立即执行一次。

那我直接给你一个标准实现作为参考。节流函数核心是控制事件触发频率，代码如下：function throttle(fn, delay, immediate = false) { let timer = null; let isImmediate = true; return function(...args) { const context = this; if (immediate && isImmediate) { fn.apply(context, args); isImmediate = false; } if (!timer) { timer = setTimeout(() => { fn.apply(context, args); timer = null; isImmediate = true; }, delay); } }; } 这个函数通过timer控制延迟执行，用isImmediate标记是否立即执行。取消功能可以通过清除timer实现，比如在返回的函数上挂载cancel方法：const throttled = throttle(fn, 1000); throttled.cancel = () => { clearTimeout(timer); timer = null; isImmediate = true; };
